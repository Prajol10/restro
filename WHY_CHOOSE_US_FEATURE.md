# Why Choose Us Feature Documentation

## Overview
This feature allows restaurant admins to manage "Why Choose Us" items that appear on the About page. Each item consists of an icon, title, description, and sort order.

## Backend Components

### 1. Model
- **File**: `backend/RestroApi/Models/WhyChooseUs.cs`
- **Fields**:
  - Id (int, primary key)
  - RestaurantId (int, foreign key)
  - Icon (string, emoji or icon representation)
  - Title (string, up to 100 characters)
  - Description (string, up to 500 characters)
  - SortOrder (int, for ordering items)
  - IsActive (bool, for soft deletion)
  - CreatedAt (DateTime, timestamp)

### 2. Database Table
- **Table**: `WhyChooseUs`
- **Schema**: 
  - Id (integer, primary key, auto-increment)
  - RestaurantId (integer, foreign key to Restaurants)
  - Icon (varchar(50))
  - Title (varchar(100))
  - Description (varchar(500))
  - SortOrder (integer, default 0)
  - IsActive (boolean, default true)
  - CreatedAt (timestamp with time zone, default UTC now)
- **Indexes**:
  - Primary key on Id
  - Foreign key constraint on RestaurantId
  - Index on RestaurantId
  - Unique index on (RestaurantId, Title)

### 3. API Endpoints
Added to `AdminController.cs`:
- `GET /api/Admin/why-choose-us` - Get all active WhyChooseUs items for the restaurant
- `POST /api/Admin/why-choose-us` - Create a new WhyChooseUs item
- `PUT /api/Admin/why-choose-us/{id}` - Update a WhyChooseUs item
- `DELETE /api/Admin/why-choose-us/{id}` - Delete a WhyChooseUs item
- `PUT /api/Admin/why-choose-us/{id}/archive` - Toggle IsActive status

Added to `RestaurantController.cs`:
- `GET /api/Restaurant/{subdomain}/why-choose-us` - Get all active WhyChooseUs items for public display

### 4. Frontend Components

#### Admin Dashboard
- **File**: `frontend/src/pages/admin/AdminDashboard.jsx`
- **New Tab**: "Why Choose Us" tab with icon 🌟
- **Features**:
  - Add new WhyChooseUs items
  - Edit existing items
  - Delete items
  - Archive/unarchive items
  - Form with icon, title, description, and sort order fields

#### Public About Page
- **File**: `frontend/src/pages/public/AboutPage.jsx`
- **Section**: "Why Choose Us" section that displays items from the database
- **Fallback**: Shows default items if none exist in database

#### Tenant Context
- **File**: `frontend/src/context/TenantContext.jsx`
- **Update**: Added fetching of WhyChooseUs data when loading restaurant data

## Database Migration
The database table was created with the following SQL:

```sql
CREATE TABLE IF NOT EXISTS "restro"."WhyChooseUs" (
    "Id" integer GENERATED BY DEFAULT AS IDENTITY,
    "RestaurantId" integer NOT NULL,
    "Icon" varchar(50) NOT NULL,
    "Title" varchar(100) NOT NULL,
    "Description" varchar(500) NOT NULL,
    "SortOrder" integer NOT NULL DEFAULT 0,
    "IsActive" boolean NOT NULL DEFAULT true,
    "CreatedAt" timestamp with time zone NOT NULL DEFAULT (now() at time zone 'utc'),
    PRIMARY KEY ("Id"),
    CONSTRAINT "FK_WhyChooseUs_Restaurants_RestaurantId" FOREIGN KEY ("RestaurantId") REFERENCES "restro"."Restaurants" ("Id") ON DELETE CASCADE
);

CREATE INDEX IF NOT EXISTS "IX_WhyChooseUs_RestaurantId" ON "restro"."WhyChooseUs" ("RestaurantId");
CREATE INDEX IF NOT EXISTS "IX_WhyChooseUs_RestaurantId_Title" ON "restro"."WhyChooseUs" ("RestaurantId", "Title");
